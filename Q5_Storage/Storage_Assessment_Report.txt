=== Storage Health Observations ===
- Root filesystem: 48% used, healthy — no immediate action needed but should be monitored as it approaches 70-80%.
- Home filesystem: only 3% used, very healthy with significant headroom.
- Inode usage is low everywhere (under 15%), so there's no risk of inode exhaustion in the near term.
- No filesystem is near capacity; the server has adequate storage for a new application deployment.

=== Recommendations for Improving Storage Utilization ===
1. Set up automated disk usage monitoring/alerts (e.g., at 80% threshold) on the root filesystem before deploying the new application server, since application logs and data can grow quickly.
2. Periodically clean up Docker images/containers on /var/lib/docker if unused, since it resides on the root partition and could consume significant space over time.
3. Consider mounting application data/logs on a separate partition (like /home or a new volume) rather than root, to isolate application storage growth from the OS partition.
4. Schedule regular log rotation to prevent unbounded log file growth from affecting available disk space.
